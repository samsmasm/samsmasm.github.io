Numerical Analysis

Overview

Bridges pure mathematics with computation. All analytical solutions are exact; all computations are approximate. Understanding when and why algorithms fail is critical for anyone using mathematics in practice.

Why this matters

Infinite-precision mathematics does not exist in a computer. The difference between a theoretically correct and a numerically stable algorithm is the difference between a model that works and one that crashes in production.

Learning outcomes
  1. 1
    Analyse floating-point arithmetic and understand sources of numerical error.
  2. 2
    Apply and compare iterative solvers for linear systems (Gauss-Seidel, conjugate gradient).
  3. 3
    Implement polynomial interpolation (Lagrange, Newton) and splines.
  4. 4
    Compute numerical derivatives and integrals with error bounds.
  5. 5
    Apply Runge-Kutta methods for ODE initial value problems; analyse stability.
  6. 6
    Understand the condition number of a matrix and its implications for linear system solving.
  7. 7
    Implement QR decomposition via Householder reflections.
⚡ Milestone checkpoint

Implement Gaussian elimination with partial pivoting. Implement RK4 and compare to Euler for a stiff ODE. Use SVD to compress an image to rank 10 and plot error vs. rank.

AI integration

Ask AI to explain why a specific scheme is unstable. Ask for a comparison of time complexity between iterative solvers.
